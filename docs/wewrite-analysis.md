## 一、WeWrite 是什么

Ray Wang (@wangray) 开源的 WeWrite 是一个 **Claude Code Skill**，用一句话让 AI 完成微信公众号文章的全流程：

**热点抓取 → 选题评分 → 大纲生成 → 写作 → SEO 优化 → AI 配图 → 排版 → 推送到草稿箱**

- GitHub：320 stars，55 forks，MIT 协议
- 单篇文章成本不到 2 毛钱（用 deepseek）
- 社区反响热烈，6.5 万浏览，636 赞

---

## 二、核心功能清单

| 功能模块 | 说明 |
|---------|------|
| 多平台热点抓取 | 从微博、头条、百度并行抓取 30 个热点话题，自动去重 |
| SEO 关键词评分 | 基于百度 + 360 搜索建议 API 的真实搜索量数据，非 LLM 猜测 |
| 结构化选题 | 生成 10 个候选题目，按 3 维度评分，与历史文章去重 |
| 5 种写作框架 | 痛点型、故事型、清单型、对比型、辣评型，各有推荐结构和钩子策略 |
| AI 痕迹去除 | 内置写作指南专门处理"去 AI 味" |
| 客户风格档案 | 每客户独立 `style.yaml`：语气、禁词、内容方向、主题、封面风格 |
| Playbook 自学习 | 对比 AI 草稿与人工编辑后的终稿，提取模式，5 次以上自动生成 playbook |
| AI 配图生成 | 封面图（2.35:1）+ 文内插图（16:9），支持豆包/DALL-E 3 |
| Markdown → 微信 HTML | 将标准 Markdown 转为微信兼容的内联样式 HTML，支持 4 套主题 |
| 直推微信草稿 | 通过微信公众号 API 直接推送到草稿箱 |
| 数据分析 | 拉取阅读/分享数据，反馈到未来选题 |

## 三、技术架构

### 技术栈

- **语言**：Python（核心脚本和工具包）
- **AI 编排**：Claude Code Skill 格式（SKILL.md 为主指令文件）
- **依赖**：markdown, beautifulsoup4, cssutils, requests, pyyaml, Pygments, Pillow
- **外部 API**：微信公众号 API、火山引擎豆包/DALL-E 3、微博/头条/百度热点、百度/360 搜索建议

### 架构模式

WeWrite 不是传统 Web 应用，而是 **Claude Code Skill** 架构：

| 传统做法 | WeWrite 做法 |
|---------|-------------|
| 建 Web App + UI + 后端 | 一个 SKILL.md + 几个 Python 脚本 |
| 用户学习新工具 | 用户在终端说一句话 |
| 数据库存状态 | YAML 文件存配置和历史 |
| 复杂部署 | 零部署，Claude Code 本地运行 |

**核心思路**：Claude 当"大脑"编排，Python 脚本当"手脚"执行，SKILL.md 当"流程图"。

### 用户流程（全自动模式）

1. 客户识别 → 读取 `clients/{name}/style.yaml`
2. 热点抓取 → 运行 `fetch_hotspots.py` 获取 30 个趋势话题
3. 历史 + SEO 数据 → 读取 `history.yaml` 去重 + `seo_keywords.py` 量化评分
4. 选题生成 → Claude 生成 10 个评分话题，自动选最高分
5. 框架选择 → 生成 5 种写作骨架，自动选最佳
6. 文章写作 → 1500-2500 字，遵循框架 + 风格配置 + playbook
7. SEO + 去 AI → 优化关键词，生成 3 个标题变体，54 字摘要
8. AI 配图 → 分析文章结构，生成封面 + 3-6 张插图
9. 排版 + 发布 → 转为微信 HTML，推送到草稿箱

每一步都有**降级回退**方案，脚本失败不阻断流程。

## 四、创新亮点

1. **Claude Code Skill 架构**：无需 Web 应用，终端一句话运行全流程
2. **Playbook 自学习系统**：从人工编辑中学习，越用越像你的风格
3. **量化 SEO 而非 LLM 猜测**：真实查询百度/360 搜索量数据
4. **微信原生适配**：内联 CSS、2.35:1 封面比例、草稿 API、中文优化配图
5. **多客户隔离**：每个客户独立目录，适合代运营场景
6. **全自动 + 优雅降级**：默认无人干预运行，每步有回退方案

## 六、对我的启发

### 启发 1：直接可用 — 拿来写公众号

技术栈完全兼容（TypeScript/Python/Claude Code），如果运营公众号：

- Fork wewrite，配置公众号 API 凭证
- 定义 `style.yaml`（语气、话题方向、禁词）
- 一句话出稿，人工微调后发布
- Playbook 系统自动学习编辑偏好

**适合场景**：技术博客、AI 工具评测、个人品牌内容

### 启发 2：方法论 — Claude Code Skill 架构模式

WeWrite 最大的创新不是"AI 写文章"，而是 **Skill 架构模式**：

- SKILL.md 定义流程编排（Claude 的"剧本"）
- Python 脚本处理 API 调用和数据处理（"手脚"）
- YAML 存配置和状态（"记忆"）
- 零部署，本地运行

这种模式可以复制到**任何重复性工作流**上。

### 启发 3：可复制的方向

| 方向 | 说明 | 复杂度 |
|------|------|--------|
| 小红书自动发文 Skill | 类似 WeWrite 但适配小红书 API/格式 | 中 |
| 闲鱼上架增强 | 参考 WeWrite 的 pipeline 模式增强现有 `/xianyu-listing` | 低 |
| Twitter/X 内容 Skill | 抓热点 → 生成推文 → 定时发布 | 中 |
| 多平台分发 Skill | 写一篇文章，自动适配公众号/小红书/知乎/Twitter 格式并分发 | 高 |
| 客户内容代运营工具 | 参考多客户架构（`clients/{name}/style.yaml`），做成可接单工具 | 高 |

## 七、建议的下一步

**最快出成果的路径**：直接 fork WeWrite，跑通一次完整流程，体验 pipeline 的威力。然后选择：

- **A）直接用** — 配置公众号开始写
- **B）改造复用** — 把 pipeline 架构应用到自己的场景（如增强闲鱼/小红书 Skill）
- **C）做新 Skill** — 参考 SKILL.md 结构，为最痛的重复工作流建自动化 pipeline
